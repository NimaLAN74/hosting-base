# Review & Approval Process  
Version 1.0

## 1. Steps
1. Draft created by author.
2. Internal technical review (Data Architect, ML Lead).
3. Comment resolution and updates.
4. Formal approval and version tagging.
5. Publication in central repo (e.g. main branch).

## 2. Versioning
- Use semantic-style versions for docs: v1.0, v1.1, etc.
- Maintain a simple changelog per major document group.
